Overview

This will implement a retrieval + QA system. The input to the system is multiple documents and a question. The output is an answer and an explanation of how the provided documents support the answer.

This pipeline comprises the following steps:

  1. Sliding window: segment the documents into passages of N (e.g., 10) sentences.
  2. Indexing: create an inverted index, i.e., a data structure that allows us to efficiently retrieve passages that contain one of more words present in the question.
  3. Retrieval: use Pyserini's BM25 to retrieve passages that are likely relevant to the question.
  4. Reranking: use monoT5 to rerank the passages retrieved in step 3. The goal of this step is to select more relevant passages to use as input to step 5.
  5. Question-answering: use a zero/few-shot language model (GPT-3.5-turbo) to answer the question based on the passages selected in step 4.
  6. Question decomposition: use a zero/few-shot language model to decompose the main question into self-contained sub-questions. Then, use the sub-questions for multi-doc QA following steps 3, 4, and 5.

This pipeline comprises the steps of the method we proposed in Visconde: Multi-document QA with GPT-3 and Neural Reranking.

In this step, we perform document segmentation to divide the input documents into smaller segments or passages. This segmentation process aids in breaking down lengthy texts into more manageable units, allowing for more efficient processing and analysis.

The segmentation is carried out using a sliding window approach, where a fixed window size is defined (e.g., 10 sentences) and moved across the document with a specified stride (e.g., 5 sentences). This approach ensures that the information within the document is adequately captured and processed while balancing the computational resources required.

This approach is particularly beneficial when dealing with large and complex documents that contain a wealth of information spread across different sections.

from tqdm import tqdm
import spacy

nlp = spacy.blank("en")
nlp.add_pipe('sentencizer')

stride = 2
max_length = 3

def window(document, stride=stride, max_length=max_length):
    segments = []
    doc_text = document['content']
    doc = nlp(doc_text)
    sentences = [sent.text.strip() for sent in doc.sents]
    for i in range(0, len(sentences), stride):
        segment = ' '.join(sentences[i:i + max_length])
        segments.append({
            "title": document['title'],
            "contents": document['title']+". "+segment,
            "segment": segment
        })
        if i + max_length >= len(sentences):
            break
    return segments

The code below performs question decomposition. It defines the question_decomposition function.

  • The question_decomposition function takes a question as input and decomposes it into self-contained sub-questions.
  • It starts by defining an instruction message template that instructs the assistant to break down a specific question into sub-questions.
  • It creates a list of messages containing alternating user and assistant messages. The user messages contain the instruction template with specific questions, and the assistant messages provide the sub-questions as responses.
  • After generating the chat conversation using the generate_chat function (OpenAI API call), the response is obtained.
  • The response is then processed using the regular expression pattern defined earlier to extract the sub-questions. Finally, the extracted sub-questions are returned as a list.
import re

def question_decomposition(question):
    """
    Decomposes the given question into self-contained sub-questions using ChatGPT.

    Args:
        question (str): The question to be decomposed.

    Returns:
        list: A list of self-contained sub-questions extracted from the original question.

    Example:
        question = "What are the advantages and disadvantages of renewable energy sources?"
        subquestions = question_decomposition(question)
        print(subquestions)
        # Output: ['What are the advantages of renewable energy sources?', 'What are the disadvantages of renewable energy sources?']
    """


    # Defining the instruction message template
    instruction = f"Please decompose the following question into self-contained sub-questions."

    # Creating a list of user and assistant messages containing the main question, demonstration examples, and the user message with the target question.
    messages = []
    messages.append({"role": "system", "content": instruction})
    for example in question_decomposition_examples:
        messages.append({"role": "user", "content": example["question"]})
        messages.append({"role": "assistant", "content": ' '.join(example["subquestions"])})

    messages.append({"role": "user", "content": question})

    # Generating the chat conversation
    response = generate_chat(messages)

    # Extracting the sub-questions using a regular expression pattern
    subquestions_pattern = re.compile(r'(?:^|\s)([^.?]+?[.?])')
    extracted_subquestions = subquestions_pattern.findall(response)

    # Cleaning and returning the sub-questions as a list
    return [sq.strip() for sq in extracted_subquestions]

import numpy as np
from sentence_transformers import util

def get_similar(question, top_k=5):
    """
    Retrieves the most similar examples from the explained training set based on the provided question using sentence transformers.

    Args:
        question (str): The input question to find similar examples for.
        top_k (int, optional): The number of similar examples to retrieve. Defaults to 5.

    Returns:
        list: A list of dictionaries representing the top-k similar examples. Each dictionary contains the following fields:
            - "context" (list): A list of contexts where each context has "text", "indices", and "passage" fields.
            - "question_links" (list): A list of question links.
            - "answer" (str): The answer to the question.
            - "question" (str): The question text.
            - "qid" (str): The unique identifier of the question.
            - "title" (str): The title of the related document.
            - "explanation" (str): An explanation providing evidence for the answer.
    """

    # Encode the input question using the SentenceTransformer model
    question_embedding = model.encode(question)

    # Calculate the cosine similarities between the question embedding and the embeddings of the explained_train dataset
    similarities = util.pytorch_cos_sim(question_embedding, explained_train_questions_embeddings).numpy().flatten()

    # Get the indices of the top-k similar examples
    top_k_indices = np.argpartition(similarities, -top_k)[-top_k:]

    # Sort the top-k indices by their similarity scores
    sorted_top_k_indices = top_k_indices[np.argsort(-similarities[top_k_indices])]

    # Retrieve the top-k similar examples from the explained_train dataset and return them as a list
    return [explained_train[index] for index in sorted_top_k_indices]

This function uses regular expressions to search the text for an "Explanation" and "Answer" section, which are expected to be formatted as "Explanation: <explanation text>\n" and "Answer: <answer text>". import re

def extract_explanation_and_answer(text):
    # Define regular expressions to match the explanation and answer
    explanation_regex = r"Explanation:\s(.+?)\n"
    answer_regex = r"Answer:\s(.+)"

    # Find the explanation and answer in the text
    explanation = re.search(explanation_regex, text).group(1)
    answer = re.search(answer_regex, text).group(1)

    return explanation, answer

The chatgpt_qa function is used to perform multi-document question answering using the ChatGPT gpt-3.5-turbo model. Similar to the InstructGPT function, the chatgpt_qa function requires a question and a list of documents as inputs.

When using the gpt-3.5-turbo model from OpenAI API for a conversational task, it is necessary to define three roles: system, user, and assistant. These roles are used to create a conversation flow between the user and the model, where the system role represents the model, the user role represents the user input, and the assistant role represents the model's response. To prompt the model, the function uses a list of messages, which includes a "role" key to specify whether the message is from the "user" or the "assistant". The first message in the list is a system message that includes the instruction on how the model should answer the question.

The function also appends the few-shot examples to the messages list, with each example being split into two messages: one from the user with the example question and documents, and one from the assistant with the example explanation and answer.

Finally, the function appends a message from the user with the target question and documents to the messages list. The generate_chat function is then called to perform the API call to the OpenAI API using the messages list as input.

The output of the generate_chat function is then post-processed using the extract_explanation_and_answer function to extract the explanation and answer provided by the model.
